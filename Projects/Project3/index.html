Unity Project – USA Summer Class (Groupwork)

About the Game

This group project explores how more human-like AI behaves in a game environment. The focus is on different detection models (vision, sound, proximity) and how enemies share information about the player.

  • Top-down shooter built in Unity.
  • Multiple AI perception systems toggled in real time.
  • Emphasis on readable AI behaviour and debugging tools (paths, rays, etc.).

Controls

  • Move: WASD
  • Shoot: Left Mouse Click

AI Modes & Debug Toggles

  • F1: Enemy is Omniscient (always knows player position)
  • F2: Enemy detects player via View Cone
  • F3: Enemy detects player via Poisson Disc sampling
  • F4: Enemy detects player via Circular Radius
  • F5: Toggle dotted path visibility to player
  • F6: Toggle enemy hearing
  • F7: Toggle Gaussian randomness on View Cone raycasts
  • F8: Toggle raycast line visibility (when Gaussian randomness is enabled)
  • F9: Toggle zoom out (view entire map)
  • F10: Restart / reset level

My Contributions

  • Implemented the shooting system, including:
    • Bullet firing logic.
    • Bullet travel behaviour.
    • Associated firing and travel sound effects.
  • Designed and implemented sound emission logic:
    • Sound attenuation based on wall material and thickness.
    • Hollow walls allow more sound to pass; solid walls block more.
  • Ensured AI blackboard data remained consistent:
    • Correct data when patrolling and chasing.
    • Enemies only share information when they have actually seen the player.
